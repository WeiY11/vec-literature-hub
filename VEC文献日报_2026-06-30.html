🎯 研究问题

车载边缘计算中，车辆高速移动导致计算任务频繁中断，传统静态卸载策略无法适应动态拓扑变化。同时，语义通信作为新兴范式，如何在RIS辅助下实现高效的任务卸载与资源分配仍缺乏系统性优化方案。现有方法在高动态场景下面临时延高、可靠性低的挑战。

🔬 解决方法

提出基于深度强化学习的混合优化框架，结合RIS与语义感知技术。利用PPO算法处理离散决策（如RIS相位配置、任务分配），结合线性规划优化连续变量（如功率分配）。该双层方法在动态车联网场景下实现自适应任务调度，显著降低平均时延。

🔄 替代方案
遗传算法(GA) 基于进化策略搜索卸载方案。优势：无需模型假设，通用性强；劣势：实时性差，大规模场景性能下降，本文方法时延降低40-50%。
量子粒子群优化(QPSO) 利用量子计算原理优化资源分配。优势：搜索空间探索能力强；劣势：参数敏感，收敛速度慢，本文方法在30辆车场景下保持高效。
博弈论方法 将卸载决策建模为非合作博弈。优势：分布式部署简单；劣势：收敛速度慢，难以保证全局最优，本文方法实现全局优化。

🔧 核心方法与技术

PPO (Proximal Policy Optimization)

近端策略优化算法，通过限制策略更新幅度保证训练稳定性。用于处理RIS相位配置、任务分配等离散决策问题，在动态环境下实现快速收敛。

RIS-Assisted Beamforming

可重构智能表面辅助波束赋形技术，通过动态调整反射相位增强信号质量。在车载场景中补偿高速移动带来的信道衰落，提升通信可靠性。

Semantic Communication

语义通信范式，提取任务数据中的语义信息而非原始比特进行传输。大幅减少通信开销，在带宽受限的车联网场景中提升传输效率。

Hybrid Optimization Framework

混合优化框架，将离散决策（PPO）与连续优化（线性规划）解耦。双层架构降低问题复杂度，实现联合任务卸载与资源分配的高效求解。

Dynamic Task Scheduling

动态任务调度机制，根据车辆位置、信道状态和计算负载实时调整卸载策略。适应车联网高动态拓扑变化，保证任务执行的连续性。

本文研究了RIS辅助的语义车载边缘计算系统中的任务卸载与资源分配问题。针对车辆高速移动导致的动态拓扑变化，提出了一种基于近端策略优化(PPO)的混合优化框架。该框架将离散决策（如RIS相位配置、任务分配）与连续优化（如功率分配）解耦，通过PPO算法处理离散变量，结合线性规划求解连续变量。仿真结果表明，所提方法相比遗传算法和量子粒子群算法，平均时延降低40-50%，且在30辆车的规模下仍能保持高效运行。

🎯 研究问题

车载网络切片需要同时支持eMBB（增强移动宽带）和URLLC（超可靠低时延通信）等不同服务质量需求的服务。传统资源管理方法缺乏可解释性，运营商难以理解AI决策逻辑，导致实际部署困难。如何在保证性能的同时提供透明、可解释的资源分配方案是关键挑战。

🔬 解决方法

提出基于可解释深度强化学习的动态资源管理框架，结合Shapley值分析与注意力机制。Shapley值量化各特征对决策的贡献度，注意力机制可视化资源分配的关注重点。该框架在实现高效资源分配的同时，提供透明的决策依据，增强运营商信任。

🔄 替代方案
传统优化方法 基于凸优化或启发式算法的资源分配。优势：理论保证最优性；劣势：难以适应动态变化，实时性差，缺乏学习能力。
黑盒DRL方法 纯深度强化学习方法如DQN、PPO。优势：自适应能力强；劣势：决策过程不透明，难以解释，运营商信任度低。
联邦学习方法 分布式训练保护数据隐私。优势：数据不出域；劣势：通信开销大，模型收敛慢，可解释性同样不足。

🔧 核心方法与技术

Explainable DRL (XAI-DRL)

可解释深度强化学习框架，在DRL基础上集成可解释性模块。通过Shapley值分析量化状态特征对动作选择的贡献，使决策过程透明化。

Shapley Value Analysis

基于博弈论的Shapley值方法，公平分配各特征对最终决策的贡献度。为每个资源分配决策提供特征重要性排序，增强可解释性。

Attention Mechanism

注意力机制可视化资源分配过程中的关注重点。通过注意力权重展示模型对不同网络切片、用户需求的关注程度，辅助运营商理解决策逻辑。

Network Slicing

网络切片技术将物理网络虚拟化为多个逻辑网络，分别服务eMBB和URLLC等不同需求。动态切片资源分配保证各类服务的QoS要求。

Multi-Objective Optimization

多目标优化框架同时考虑频谱效率、时延、可靠性等多个指标。通过加权求和或帕累托优化平衡不同目标，满足多样化服务需求。

⭐⭐⭐⭐⭐ 首次将可解释AI引入车载网络切片资源管理，解决AI决策黑盒问题
⭐⭐⭐⭐ 结合Shapley值与注意力机制，提供多维度的决策解释
⭐⭐⭐⭐ 同时支持eMBB和URLLC服务，验证了框架的多场景适用性
⭐⭐⭐ 为可解释AI在通信网络中的应用提供了新范式

🎯 研究问题

下一代无人机网络中，无人机作为空中边缘节点需要提供计算和通信服务。然而，无人机受限于电池容量和计算资源，如何在保证覆盖范围的同时优化功率和带宽分配是关键挑战。传统方法难以适应动态的用户分布和环境变化。

🔬 解决方法

提出FLARE框架，利用多智能体强化学习优化无人机位置部署和资源分配。通过聚类算法将用户分组，每个无人机智能体独立学习最优策略。混合强化学习结合价值方法和策略梯度，实现功率、带宽和位置的联合优化，覆盖用户数提升约73%。

🔄 替代方案
单智能体方法 集中式控制所有无人机。优势：全局最优；劣势：通信开销大，可扩展性差，单点故障风险高。
启发式部署 基于规则或几何方法的无人机部署。优势：计算简单；劣势：无法适应动态变化，资源利用率低。
纯价值方法(DQN) 基于值函数的强化学习。优势：训练稳定；劣势：难以处理连续动作空间（如位置优化），本文混合方法更优。

🔧 核心方法与技术

Multi-Agent RL

多智能体强化学习框架，每个无人机作为独立智能体学习最优策略。分布式部署降低通信开销，提高系统可扩展性和鲁棒性。

Clustering-Based Grouping

基于聚类的用户分组算法，将地理分布的用户划分为多个簇。每个无人机负责一个簇的服务，优化覆盖效率和资源分配。

Hybrid RL Approach

混合强化学习方法，结合价值方法（处理离散决策）和策略梯度（处理连续动作）。实现功率、带宽和位置的联合优化。

Dynamic Positioning

动态位置优化机制，根据用户分布实时调整无人机位置。在保证覆盖的同时最小化能耗，适应动态环境变化。

Resource Allocation

联合功率和带宽分配算法，根据各簇用户需求动态调整资源。最大化系统吞吐量，保证服务质量公平性。

⭐⭐⭐⭐ 提出FLARE框架，将多智能体强化学习应用于无人机网络资源优化
⭐⭐⭐⭐ 设计聚类+混合RL的两阶段方法，有效解决连续-离散混合优化问题
⭐⭐⭐⭐ 覆盖用户数提升约73%，验证了方法的有效性
⭐⭐⭐ 为无人机辅助的边缘计算提供了新的资源管理思路

🎯 研究问题

车联网(IoV)中，计算任务通常具有有向无环图(DAG)结构，子任务间存在依赖关系。传统任务卸载方法将任务视为独立单元，忽略子任务间的时序和依赖约束。同时，车辆高速移动导致网络拓扑动态变化，边缘节点计算能力异构，如何在满足任务依赖约束的同时优化卸载决策和资源配置是关键挑战。

🔬 解决方法

提出DVTP框架，结合变分图注意力网络(VGAT)和Transformer进行DAG任务表示学习，提取任务的时空特征。利用深度强化学习算法进行任务调度决策，联合优化卸载策略、计算资源和通信资源分配。VGAT编码DAG拓扑结构，Transformer捕捉全局依赖关系，两者协同实现高效的DAG任务调度。

🔄 替代方案
贪心启发式算法 基于优先级规则的贪心策略。优势：计算简单、实时性好；劣势：忽略全局优化，性能次优，难以处理复杂DAG结构。
传统DRL方法(DQN/PPO) 不使用图表示学习的强化学习。优势：自适应能力强；劣势：难以捕捉DAG拓扑结构和子任务依赖关系。
Lyapunov优化 基于Lyapunov理论的在线优化方法。优势：理论保证稳定性；劣势：需要精确系统模型，难以处理动态拓扑。

🔧 核心方法与技术

DVTP Framework

DAG任务调度框架，集成变分图注意力网络和Transformer。VGAT编码DAG拓扑结构提取局部特征，Transformer捕捉全局依赖关系，两者协同实现DAG任务的高效表示。

Variational Graph Attention Network (VGAT)

变分图注意力网络，通过注意力机制聚合DAG中邻居节点信息。学习子任务的嵌入表示，捕捉任务间的依赖关系和时序约束。

Transformer Encoder

Transformer编码器提取DAG任务的全局特征。通过自注意力机制建模长距离依赖，补充VGAT的局部特征提取能力。

Deep Reinforcement Learning

深度强化学习算法进行任务调度决策。智能体根据DAG任务表示学习结果，动态决定子任务的卸载目标和资源分配方案。

Joint Resource Allocation

联合资源分配策略，同时优化计算频率、传输功率和带宽分配。满足DAG任务依赖约束的同时最小化完成时间。

主动推理框架，基于自由能最小化原理进行感知-行动决策。设备通过内部生成模型预测环境状态，选择最小化预期自由能的动作，实现智能卸载。

Generative Model Planning

基于生成模型的规划机制，构建环境的内部表征。通过模拟未来状态预测不同卸载策略的效果，实现前瞻性决策。

Free Energy Minimization

自由能最小化原理，平衡推理精度(认识论价值)和计算成本(实用价值)。为卸载决策提供统一的优化目标。

6G Symbiotic IoT Architecture

6G共生物联网架构，终端-边缘-云三级协同。各层级部署不同规模LLM，根据任务需求动态分配推理资源。

Proactive Offloading

主动式卸载策略，基于环境预测提前做出决策。相比被动响应，减少因网络波动导致的推理中断和时延增加。

🎯 研究问题

仅依赖云端或边缘部署高级语言模型都存在不足：云端部署时延高、成本高，边缘部署受资源限制、能力有限。如何实现边缘SLM与云端LLM的高效协同，在隐私、成本、时延、定制化等多目标间取得平衡，是关键挑战。

🔬 解决方法

系统性综述边缘SLM与云端LLM协同方法。推理方面，按任务分配、任务分割、混合协同分类；训练方面，涵盖参数对齐、剪枝、双向蒸馏、小模型引导优化等分布式适应技术。同时讨论评估基准、数据集、隐私保护方法和垂直应用，为边云智能协同建立基础框架。

🔄 替代方案
纯云端部署 所有推理和训练在云端完成。优势：模型能力最强；劣势：时延高、成本高、隐私风险大。
纯边缘部署 在边缘设备部署完整模型。优势：低时延、数据本地；劣势：受资源限制，模型能力有限。
模型压缩 通过量化、剪枝压缩大模型部署到边缘。优势：单设备完整功能；劣势：精度损失，难以保持大模型能力。

🔧 核心方法与技术

Task Assignment

任务分配策略，根据查询复杂度决定由边缘SLM或云端LLM处理。简单查询本地处理，复杂查询上传云端，平衡时延与精度。

Task Splitting

任务分割方法，将单个查询拆分到边云协同处理。边缘处理输入编码，云端处理复杂推理，结果融合，降低通信开销。

Mixture-Based Collaboration

混合协同机制，边缘SLM与云端LLM动态协作。通过知识蒸馏、提示优化等技术，实现能力互补。

Parameter Alignment

参数对齐技术，使边缘SLM与云端LLM的知识保持一致。通过联邦学习或蒸馏，实现知识同步。

Bidirectional Distillation

双向蒸馏机制，云端LLM向边缘SLM蒸馏知识，边缘SLM向云端反馈本地数据分布。实现双向知识流动，提升整体性能。
