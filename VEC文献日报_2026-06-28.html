PPO-Based Hybrid Optimization for RIS-Assisted Semantic Vehicular Edge Computing

🎯 研究问题

车载边缘计算中，车辆高速移动导致信道快速变化，传统基于比特的通信方式在资源受限场景下效率低下。语义通信虽可提取关键信息减少传输量，但在动态车联网环境中，如何联合优化卸载比例、语义符号数量和RIS相位偏移以最小化时延，是一个高维非凸的复杂优化问题。

🔬 解决方法

提出基于RIS辅助的语义车载边缘计算架构，设计双层优化框架：上层利用PPO（近端策略优化）处理离散决策（卸载比例、语义符号数量），下层用线性规划求解连续变量（RIS相位偏移）。两层交替迭代，在动态环境中实现端到端时延最小化。

🔄 替代方案
遗传算法(GA) 将联合优化建模为进化搜索问题。优势：不要求问题凸性，全局搜索能力强；劣势：计算复杂度高，难以满足车联网实时性要求。
交替优化(AO) 固定部分变量交替优化其余变量。优势：收敛性有理论保证；劣势：容易陷入局部最优，在高维场景性能有限。
多智能体RL(MARL) 将各车辆视为独立智能体协同决策。优势：分布式部署，可扩展性好；劣势：训练不稳定，通信开销大。

🔧 核心方法与技术

RIS-Assisted Semantic VEC 构建RIS辅助的语义车载边缘计算架构，利用可重构智能表面增强信道质量，结合语义通信提取关键信息特征，减少传输数据量，从物理层提升系统效率。
PPO for Discrete Decision 利用近端策略优化算法处理离散决策变量（任务卸载比例和语义符号数量），通过策略裁剪机制保证训练稳定性，适应动态车联网环境。
LP for Continuous Optimization 在给定离散决策后，利用线性规划高效求解RIS相位偏移等连续变量，降低计算复杂度，实现快速收敛。
Dual-Level Alternating Framework 设计双层交替迭代框架，上层RL智能体输出离散决策，下层LP求解器优化连续变量，两层相互反馈直至收敛。
Scalability Design 算法设计考虑可扩展性，在30辆车同时活跃的高密度场景下仍保持低时延，验证了框架在大规模网络中的有效性。

🎯 研究问题

车载网络切片需要同时服务eMBB（高带宽）和URLLC（低时延高可靠）两类差异化业务，传统资源分配方法难以在动态信道条件下满足多样化QoS需求。更关键的是，基于深度强化学习的黑盒决策缺乏可解释性，阻碍了其在安全关键的车载场景中的实际部署。

🔬 解决方法

提出XRL框架，在近乎实时的RAN控制器上运行，将Shapley值与注意力机制结合，为强化学习决策提供可解释性。Shapley值量化各特征对决策的贡献度，注意力机制增强关键状态特征的权重，从而提升URLLC和eMBB的QoS满足率。

🔄 替代方案
Lyapunov在线优化 利用Lyapunov漂移加小方法实现无需预测的在线资源分配。优势：有严格的队列稳定性和时延界保证；劣势：需要精确的系统模型，对非线性约束处理能力有限。
博弈论方法 将网络切片资源竞争建模为非合作博弈。优势：分布式决策，适合多运营商场景；劣势：纳什均衡存在性和效率损失难以保证。
图神经网络(GNN) 利用车辆拓扑图结构进行资源分配。优势：天然适配网络拓扑动态变化；劣势：训练数据需求大，实时推理开销较高。

🔧 核心方法与技术

XRL Framework 提出可解释强化学习框架，在近实时RAN控制器上运行，将Shapley值分析与RL决策结合，实现透明的资源分配决策过程。
Shapley Value Analysis 利用Shapley值量化每个状态特征对RL决策的边际贡献，为网络切片资源分配提供博弈论基础的可解释性。
Attention Mechanism 引入注意力机制动态加权关键状态特征，增强RL策略对重要信道条件和QoS指标的敏感度。
Near-Real-Time RIC 基于O-RAN架构的近实时无线接入网控制器实现，决策时延满足车载场景的毫秒级响应需求。
Multi-Slice QoS Management 同时管理eMBB和URLLC两类网络切片的资源分配，URLLC QoS满足率达80.13%，eMBB达73.21%。

⭐⭐⭐⭐ 首次将Shapley值+注意力机制引入车载网络切片的RL决策，实现可解释的资源管理

🎯 研究问题

下一代无人机网络中，多无人机协同为用户提供边缘计算服务时面临两大挑战：一是无人机位置动态变化导致覆盖区域不断改变，二是有限的电池和计算资源限制了服务能力。如何在无人机移动性约束下联合优化位置部署和资源分配，是一个复杂的非凸优化问题。

🔬 解决方法

提出FLARE框架，先用Silhouette-based K-Means聚类对用户分组，再用混合强化学习策略（结合MADDPG和DDPG）联合优化无人机位置和发射功率。多智能体协同决策，实现用户覆盖和服务质量的最大化。

🔄 替代方案
单智能体RL 用单个DDPG智能体集中控制所有无人机。优势：全局最优性好；劣势：状态空间随无人机数量指数增长，扩展性差。
启发式部署 基于几何中心或Voronoi图的无人机位置部署。优势：计算简单，实时性好；劣势：无法联合优化功率分配，性能次优。
合同理论 用合同理论激励无人机参与协作。优势：经济学基础，激励兼容；劣势：信息不对称假设限制了实际应用。

🔧 核心方法与技术

Silhouette-K-Means Clustering 基于轮廓系数的K-Means聚类算法动态分组移动用户，确定最优无人机数量和初始覆盖区域划分。
MADDPG 多智能体深度确定性策略梯度算法，每架无人机作为独立智能体，在集中训练分散执行框架下学习协同策略。
Hybrid RL Strategy 混合MADDPG和DDPG的强化学习策略，在协同决策和独立优化之间取得平衡，提升收敛速度和稳定性。
Joint Position-Power Optimization 联合优化无人机三维位置和发射功率，在覆盖范围和能耗之间取得帕累托最优。
Dynamic User Serving 动态适应用户分布变化，实时调整无人机部署策略，相比基准方法提升73.45%的服务用户数。

🔄 替代方案
传统LS信道估计 最小二乘法信道估计，简单直接。优势：实现简单，无需先验知识；劣势：在FA端口移动场景下精度差，导频开销大。
标准MADDPG 直接用多智能体RL联合优化所有变量。优势：端到端学习，无需手动分解问题；劣势：状态空间过大，训练困难，收敛慢。
块坐标下降(BCD) 分块优化端口选择、功率控制和卸载决策。优势：每块问题简化，有收敛保证；劣势：块间耦合强，交替优化性能受限。

🔧 核心方法与技术

IBM-CCS 信息瓶颈度量增强的信道压缩感知方法，通过整合信息相关性有效捕获FA信道的关键特征，提升端口移动场景下的信道估计精度。
HiTDMA 博弈论辅助的分层孪生决斗多智能体算法，分层结构有效解耦和协调任务，博弈论降低功率控制变量维度。
Game Theory Dimensionality Reduction 利用博弈论将功率控制变量维度降低，使DRL智能体能够更高效地达到优化目标，提升训练效率和收敛性。
Joint Port-Power Optimization 联合优化FA端口选择和发射功率控制，在信道估计不完美条件下实现系统时延最小化。
Robust Offloading Design 考虑不完美CSI的鲁棒卸载设计，在不同端口密度下均保持良好的估计精度和卸载性能。

🎯 研究问题

联邦学习是边缘智能的重要范式，但现有工作通常假设共享模型是"完美"的。实际部署中，模型不可避免地受到各种扰动（量化误差、传输噪声、对抗攻击等），导致性能严重下降。如何提升联邦模型对扰动的鲁棒性，是一个亟待解决的问题。

🔬 解决方法

提出SMRFL方法，从模型景观（loss landscape）的几何特性出发，通过min-max优化促使模型收敛到"平坦极小值"。在平坦极小值附近，即使模型参数受到扰动，损失值仍然保持较低水平，从而天然具备鲁棒性。理论证明SMRFL可与无扰动FL保持相同收敛速率。

🔄 替代方案
对抗训练(AT) 在训练中加入对抗样本增强鲁棒性。优势：针对性强，对特定攻击有效；劣势：计算开销大，泛化到未知扰动类型的能力有限。
模型蒸馏 用大模型指导小模型训练，提升泛化能力。优势：模型轻量化，部署简单；劣势：需要额外的大模型支持，通信开销增加。
正则化方法 通过权重裁剪、dropout等正则化技术提升鲁棒性。优势：实现简单，几乎无额外计算开销；劣势：鲁棒性提升有限，缺乏理论保证。

🔧 核心方法与技术

SMRFL Sharpness-Aware Minimization-based Robust Federated Learning，通过min-max优化寻找平坦极小值，使模型在参数扰动下仍保持低损失。
Flat Minimum Optimization 最小化模型参数邻域内的最大损失值，促使模型收敛到平坦极小值区域，从根本上降低对扰动的敏感性。
Min-Max Optimization 将鲁棒FL建模为min-max问题：内层最大化邻域扰动，外层最小化最坏情况损失，形成对抗训练的理论框架。
Convergence Guarantee 理论证明SMRFL可在与无扰动FL相同的速率下收敛，为方法的有效性提供严格数学保证。
Multi-Perturbation Robustness 在三种扰动场景（量化、噪声、攻击）和两个真实数据集上，相比三种基准方法显著提升鲁棒性。

🎯 研究问题

在私有设备数据上微调大语言模型是实现个性化AI的关键，但边缘设备面临三大挑战：计算资源不足、设备异构性强、数据分布不均衡。传统联邦学习方法（如FedLoRA）要求每台设备执行反向传播，对资源受限设备负担过重。

🔬 解决方法

提出SplitFrozen框架，将LLM分为设备端冻结层（仅执行前向传播）和服务端微调层（集成LoRA进行参数高效微调）。设备端无需反向传播，大幅降低计算负担。配合流水线并行策略解耦设备-服务器计算，加速训练过程。

🔄 替代方案
FedLoRA 每台设备本地执行LoRA微调后上传参数。优势：数据隐私保护好；劣势：设备端仍需反向传播，资源消耗大。
SplitLoRA 将LoRA层拆分到设备和服务端协同计算。优势：降低设备计算量；劣势：设备端仍需部分反向传播，异构设备适配性差。
全参数云端协同微调 将全部模型参数放在云端，设备仅传输梯度。优势：设备零计算负担；劣势：通信开销极大，隐私风险高。

🔧 核心方法与技术

Device-Side Freezing 设备端模型层完全冻结，仅执行前向传播，无需反向传播，将设备计算负担降至最低。
Server-Side LoRA 服务端集成低秩适应（LoRA）进行参数高效微调，仅更新少量低秩矩阵，大幅降低训练成本。
Pipeline Parallelism 流水线并行策略解耦设备和服务器计算，通过分解的反向传播实现异步训练，加速整体流程。
Heterogeneous Device Support 不同资源受限设备只需执行前向传播，无需适配各自计算能力，天然支持设备异构。
Data Skewness Handling 在极端数据不均衡（Non-IID）场景下，模型准确率比FedLoRA和SplitLoRA提升69.4%。

6G边缘智能要求通过联邦学习在异构边缘设备上协作训练AI模型，但面临双重挑战：一是参数共享可能导致数据重建攻击，隐私泄露风险高；二是统一的全局模型难以适应各设备本地数据分布的差异（Non-IID）。现有方法在隐私保护和个性化之间难以兼顾。

🔬 解决方法

提出DPFNAS框架，将个性化差分隐私与自适应神经架构搜索结合。利用样本级表征进行知识共享，应用个性化DP策略抵抗重建攻击；同时设计隐私感知的NAS算法，为每个设备生成定制化的网络架构和超参数，在隐私约束下实现分布感知的适配。

🔄 替代方案
PerFedRLNAS 现有的个性化联邦NAS方法。优势：支持个性化架构搜索；劣势：缺乏差分隐私保护，易受重建攻击，且模型尺寸较大。
FedAvg + DP-SGD 在标准联邦学习中加入DP-SGD噪声。优势：隐私保证严格；劣势：全局统一架构无法适配异构设备，模型精度下降明显。
知识蒸馏+DP 用知识蒸馏替代参数共享，加入差分隐私。优势：减少参数传输；劣势：蒸馏过程引入额外误差，在Non-IID场景性能下降。

Representation-Based FL 利用样本级表征（而非模型参数）进行知识共享，从源头降低数据重建攻击的风险。
Personalized DP Strategy 个性化差分隐私策略，为每个设备定制隐私预算分配，在隐私保护和模型性能间取得最优平衡。
Privacy-Aware NAS 隐私感知的神经架构搜索算法，在差分隐私约束下为每个设备生成定制化的网络架构和超参数。
Convergence Guarantee 首个为基于表征的联邦学习提供理论收敛保证的个性化DP方案，数学证明收敛性。

🎯 研究问题

将推荐系统重排模型部署到移动设备需要模型压缩，但现有低比特量化方法忽略了用户个性化需求——同一压缩模型无法适配不同用户偏好。而本地微调又消耗设备电量和算力。如何在设备-云端协同下实现个性化且资源高效的模型压缩，是核心挑战。

🔬 解决方法

提出CHORD框架，利用混合比特宽度量化压缩模型，并通过云端超网络为每个用户生成个性化参数，无需设备端梯度更新。超网络根据用户特征动态生成不同量化层的参数，实现"一次压缩、千人千面"的个性化适配，同时大幅降低传输和计算开销。

🎯 研究问题

LLM具备多任务、多模态、多对话能力，但部署面临两难：本地设备资源不足（计算、内存、能耗），云端部署无法保证实时性且通信/使用成本高。如何为每个任务/对话动态决策推理位置（本地vs云端）和模态选择，以最大化长期回报（质量、时延、成本），是核心挑战。

🔬 解决方法

设计TMO系统，包含轻量级本地LLM（处理简单任务）和大规模云端LLM（处理多模态复杂任务）。开发资源约束强化学习（RCRL）策略，为每个任务/对话动态优化推理位置和多模态数据源选择，在资源约束下最大化长期回报（响应质量+低时延+低成本）。

🔄 替代方案
固定路由策略 基于规则将所有任务固定分配到本地或云端。优势：实现简单，零决策开销；劣势：无法适应用例动态变化，资源利用效率低。
LLM-as-Agent 用LLM自身作为智能体决策推理位置。优势：语义理解能力强；劣势：决策本身需要额外推理开销，循环依赖问题。
模型分割推理 将模型前半部分放本地、后半部分放云端，协同推理。优势：减少通信量；劣势：需要频繁中间结果传输，对网络时延敏感。

🔧 核心方法与技术

TMO System Three-M Offloading系统，支持多模态、多任务、多对话的本地-云端LLM推理卸载决策。
Lightweight Local LLM 轻量级本地LLM处理简单任务，保证低时延和隐私；大规模云端LLM处理复杂多模态任务。
RCRL Strategy 资源约束强化学习策略，在设备资源限制下动态优化推理位置和多模态数据源选择。
M4A1 Dataset 构建新数据集M4A1，包含跨模态、任务、对话和LLM配置的回报和成本指标，支持卸载决策评估。
Long-Term Reward Optimization 优化长期回报（响应质量+时延+成本），而非单次决策，实现全局最优的推理调度。
