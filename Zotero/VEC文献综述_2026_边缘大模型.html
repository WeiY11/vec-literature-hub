LLM-empowered Cooperative Content Caching in Vehicular Fog Caching-assisted Platoon Networks

👥 Bowen Tan, Qiong Wu, Pingyi Fan, Kezhi Wang, Nan Cheng, Wen Chen
📖 arXiv Preprint 📅 2026-02-04 🔗 10.48550/arXiv.2602.04471
🔍 深度分析
🎯 研究问题

车队网络中内容缓存面临高速移动性带来的内容需求快速变化和网络拓扑动态重构的双重挑战。传统启发式缓存策略缺乏自适应性，难以应对多源异构信息的综合决策；基于强化学习的方法泛化能力有限，在新场景下需要频繁重训练，无法满足车队场景的实时性要求。

🔬 解决方法

提出LLM驱动的三层协同缓存架构（本地车辆→VFC集群→云服务器），利用大语言模型的上下文推理能力综合处理用户画像、历史请求、内容特征等多源信息进行实时缓存决策。通过专用提示工程框架将缓存约束编码为结构化Prompt引导LLM输出可行策略，结合层次化确定性映射实现跨层精准内容放置，且基于LLM的零样本泛化能力无需重训练即可适应动态环境。

🔄 替代方案
深度强化学习(DRL) 基于DQN或PPO的缓存决策，训练后可实现快速推理，但泛化能力弱，拓扑变化需重训练，且难以处理多源异构输入信息。
GNN+联邦学习协同缓存 利用图神经网络建模车辆间拓扑关系，结合联邦学习实现分布式协同，隐私保护好但通信开销大且对动态拓扑适应性有限。
基于内容流行度预测的缓存 通过时序模型（如LSTM/Transformer）预测内容流行度进行预缓存，实现简单但难以捕捉用户个性化需求和跨层协同。
🔧 核心方法
  • 三层缓存架构 提出面向 VFC 辅助车队的本地车辆 → 动态 VFC 集群 → 云服务器三级协同缓存体系，实现存储资源的分层调度与跨域协作
  • LLM 智能决策引擎 将大语言模型引入缓存决策环节，实时处理用户画像、历史请求、内容特征和动态系统状态等多源异构信息，替代传统启发式规则
  • Prompting 框架设计 设计专用提示工程框架，将任务目标（命中率最大化）与缓存约束（容量/时延/一致性）编码为结构化 Prompt，引导 LLM 输出可行缓存策略
  • 层次化确定性映射 提出 Hierarchical Deterministic Caching Mapping 策略，将 LLM 的预测结果映射到三级缓存层的确定性放置方案，保证自适应请求预测与精准内容部署
  • 免重训练适配 整体方案基于 LLM 的上下文推理能力，无需针对新场景频繁微调，即可适应车流拓扑和内容分布的动态变化
⭐ 创新贡献
  • ★★★★★ 首次将 LLM 引入车联网雾缓存场景，开创性地用大模型替代传统优化/强化学习进行缓存决策，思路新颖
  • ★★★★☆ 三层协同缓存架构贴合实际车队网络拓扑（本地+VFC+云），层次划分合理且具有工程可行性
  • ★★★★☆ 提示工程框架将缓存约束结构化编码的方法具有通用性，可迁移到其他边缘资源管理问题
  • ★★★☆☆ 当前仅为 arXiv 预印本，实验验证的充分性和大规模仿真结果有待后续期刊版本补充
📝 中文摘要
展开 / 收起摘要

本文针对车队网络中的内容缓存问题，提出了一种基于大语言模型（LLM）的三层协同缓存架构。在车队行驶过程中，附近车辆可形成车载雾缓存（VFC）集群，为车队成员提供额外的存储与计算资源。该方案将缓存空间战略性地协调分配至三个层级：本地车队车辆、动态 VFC 集群以及远端云服务器。核心创新在于引入 LLM 进行实时智能缓存决策——LLM 能够综合处理用户画像、历史访问数据、内容特征及动态系统状态等多源异构信息。作者设计了专门的提示工程框架，将缓存任务目标和容量、时延等约束条件编码为结构化提示输入。在此基础上，提出了层次化确定性缓存映射策略，实现对用户请求的自适应预测和跨三层的精准内容放置，且无需频繁重训练模型即可适应动态网络环境。

COMLLM: Multi-turn Reasoning LLMs for Task Offloading in Mobile Edge Computing

👥 Ning Yang, Chuangxin Cheng, Haijun Zhang
📖 arXiv Preprint 📅 2026-04-08 🔗 10.48550/arXiv.2604.07148
🔍 深度分析
🎯 研究问题

移动边缘计算中的任务卸载需要在时延约束下做出动态序列决策，但传统启发式方法缺乏自适应性，深度强化学习(DRL)泛化能力有限且网络拓扑变化时需重新训练。LLM虽具备语义推理能力，但标准监督微调(SFT)容易产生短视策略，无法感知当前决策对未来系统状态的长期影响。

🔬 解决方法

提出COMLLM框架，融合GRPO（组相对策略优化）与LACS（前瞻协作仿真）机制。GRPO通过组内相对奖励信号替代SFT训练LLM生成卸载策略，LACS通过多步Monte Carlo仿真联合建模服务器队列动态，使模型感知当前决策的长期影响。多轮推理框架将卸载建模为序列决策过程，并展现出零样本拓扑泛化能力——小规模网络训练的模型可直接应用于更大规模未见拓扑。

🔄 替代方案
GNN+DRL拓扑感知卸载 利用图神经网络编码网络拓扑，结合DRL进行卸载决策，拓扑建模能力强但泛化性有限，仍需针对不同规模网络重新训练。
多智能体强化学习(MARL) 各边缘节点作为独立智能体协同决策，支持分布式执行但训练稳定性差且通信开销大，难以实现全局最优。
LLM Chain-of-Thought推理 利用大语言模型的思维链能力逐步推理卸载策略，语义理解能力强但缺乏长期规划能力，推理延迟较高。
🔧 核心方法
  • GRPO 强化学习 采用 Group Relative Policy Optimization（GRPO）替代传统 SFT 微调，通过组内相对奖励信号训练 LLM 生成卸载策略，避免监督学习中的短视决策问题
  • LACS 前瞻仿真 提出 Look-Ahead Collaborative Simulation（LACS）机制，在训练阶段通过多步 Monte Carlo 仿真联合建模服务器队列动态，使模型感知当前决策对未来系统状态的长期影响
  • 多轮推理框架 将任务卸载建模为多轮序列决策过程，LLM 在每个时步基于当前观测进行推理并输出卸载动作，实现闭环在线决策
  • 零样本拓扑泛化 实验证明模型在小规模网络上训练后，可直接泛化到更大规模的未见拓扑结构，无需重新训练——解决了 DRL 方法拓扑变化需重训练的核心痛点
⭐ 创新贡献
  • ★★★★★ GRPO + LACS 组合首次解决了 LLM 用于 MEC 卸载时的"短视策略"问题，使模型具备长期规划能力，方法论贡献突出
  • ★★★★★ 零样本拓扑可扩展性是 MEC 卸载领域的重大突破，直接解决了传统 DRL 在网络拓扑变化时需重训练的瓶颈
  • ★★★★☆ 多轮推理框架将 LLM 的语义推理能力与边缘计算决策有效结合，为"LLM as Agent"在边缘系统中的应用提供了范例
  • ★★★★☆ 在时延优化和负载均衡公平性两个指标上均接近最优，实验结果具有说服力
📝 中文摘要
展开 / 收起摘要

计算密集型应用对移动设备的时延性能提出了严苛要求。移动边缘计算（MEC）通过任务卸载来缓解这一问题，但传统启发式方法缺乏自适应性，深度强化学习（DRL）方法泛化能力有限且在拓扑变化时需要重新训练。大语言模型（LLM）具备语义推理能力，但标准的监督微调（SFT）容易产生短视策略。为此，本文提出 COMLLM——一个融合 GRPO（组相对策略优化）与前瞻协作仿真（LACS）机制的生成式框架。LACS 通过多步蒙特卡洛推演联合建模服务器队列动态，使模型能捕捉当前决策对未来系统状态的长期影响。实验结果表明，COMLLM 在时延优化和负载均衡公平性方面均接近最优水平。特别值得注意的是，该模型展现出零样本拓扑可扩展性——在小规模网络上训练的模型可直接泛化到更大规模的未见网络拓扑，无需任何重训练。

LLM MEC Task Offloading GRPO Multi-turn Reasoning Reinforcement Learning

Foundation Model Empowered Synesthesia of Machines (SoM): AI-Native Intelligent Multi-Modal Sensing-Communication Integration

👥 Xiang Cheng, Boxun Liu, Xuanyu Liu, Ensong Liu, Ziwei Huang
📖 IEEE Transactions on Network Science and Engineering IF ≈ 6.6 📅 2026 🔗 10.1109/TNSE.2025.3587238
🔍 深度分析
🎯 研究问题

面向6G网络的感知-通信一体化系统（SoM）需要将多模态感知与通信深度融合，但现有设计依赖任务专用AI模型，泛化能力有限，难以适应多样化的无线场景需求。信道建模、收发机设计和协作感知等各环节独立优化，缺乏统一的智能框架来协同全链路功能。

🔬 解决方法

提出"机器联觉"(SoM)概念，系统性构建双路线基础模型框架：通用LLM路线（LLM4PG/LLM4SG/LLM4WM）将语言模型的生成能力创造性迁移到信道参数生成和无线多任务处理；领域专用路线（WiCo/WiFo/WiPo）构建覆盖信道建模、收发机设计到协作感知全链路的无线基础模型系列，通过大规模预训练突破任务专用模型的性能瓶颈。

🔄 替代方案
纯LLM跨域迁移方案 仅利用通用大语言模型进行无线参数生成，无需训练领域专用模型，成本低但领域适配性弱，精度受限于LLM对无线物理层的理解深度。
端到端深度学习收发机 使用Autoencoder等架构进行收发机联合优化，针对特定任务效果好但泛化能力有限，不同场景需要重新训练。
传统数学建模+优化求解 基于经典信道模型和优化理论进行系统设计，理论保证强但难以处理复杂的非线性场景和高维参数空间。
🔧 核心方法
  • SoM 系统框架 提出"机器联觉"（Synesthesia of Machines）概念，构建 AI 原生的智能多模态感知-通信一体化系统框架，面向 6G 网络设计
  • 双路线基础模型 系统性地将基础模型分为两大类：通用基础模型（LLMs）和 SoM 领域专用基础模型（无线基础模型），并分别提出两条技术路线
  • LLM4PG / LLM4SG 利用 LLM 生成路径损耗模型（LLM4PG）和散射体模型（LLM4SG），将语言模型的生成能力引入无线信道建模
  • WiCo / WiFo / WiPo 提出系列无线基础模型：WiCo（无线信道基础模型）、WiFo（无线基础模型）、WiPo（无线协作感知基础模型），覆盖信道建模、收发机设计和多节点协作感知
  • LLM4WM 提出基于 LLM 的无线多任务收发机（LLM4WM），利用 LLM 的统一架构同时处理多种无线信号处理任务
⭐ 创新贡献
  • ★★★★★ 首次系统性地将基础模型（FM）引入无线感知-通信一体化系统，提出了完整的分类框架和双路线方法论，具有开创性意义
  • ★★★★★ 提出的无线基础模型系列（WiCo/WiFo/WiPo）覆盖了从信道建模到协作感知的全链路，为 6G 通信系统设计提供了全新范式
  • ★★★★☆ LLM4PG/LLM4SG 将 LLM 的文本生成能力创造性地迁移到无线信道参数生成，展示了跨域迁移学习的巨大潜力
  • ★★★★☆ 发表于 IEEE TNSE（IF≈6.6），实验充分验证了 FM 相比任务专用模型的显著优越性，学术质量有保障
📝 中文摘要
展开 / 收起摘要

本文面向 6G 网络提出了"机器联觉"（SoM）概念——一种 AI 原生的智能多模态感知-通信一体化框架。现有 SoM 设计依赖于任务专用 AI 模型，泛化能力有限。本文首次系统性地将基础模型（FM）引入 SoM 系统设计，提出了两大技术路线：基于通用基础模型（LLM）的路线和基于 SoM 领域专用基础模型（无线基础模型）的路线。在 LLM 路线中，提出了 LLM4PG（路径损耗生成）、LLM4SG（散射体生成）和 LLM4WM（无线多任务收发机）三个创新模块。在无线基础模型路线中，提出了 WiCo（无线信道基础模型）、WiFo（无线基础模型）和 WiPo（无线协作感知基础模型）。大量实验结果验证了基础模型在信道建模精度、收发机性能和协作感知效率等方面相比传统任务专用模型的显著优势。

Performance Analysis of Multi-UAV-aided NB-IoT Communication System

👥 Siva Rama Krishna M., Naveen Mysore Balasubramanya
📖 IEEE Transactions on Mobile Computing IF ≈ 8.3 CCF-A 📅 2026 🔗 10.1109/TMC.2026.3673845
🔍 深度分析
🎯 研究问题

偏远地区或基础设施缺失场景下的大规模IoT设备接入是非地面网络(NTN)面临的核心挑战。3GPP已将NB-IoT确定为NTN-IoT候选技术，但单UAV覆盖有限、OMA调度接入效率低，而NOMA虽能提高频谱效率但静态分组无法适应动态流量变化，亟需系统性的多UAV协同NB-IoT性能分析与优化方案。

🔬 解决方法

构建多UAV中继NB-IoT系统，系统性对比OMA/NOMA调度策略，提出基于LTTF（最长处理时间优先）的NOMA动态分组策略，根据实时负载动态调整用户分组。在三种流量模型（离散均匀、泊松、Beta分布）下推导了平均传输时延的闭合式解析表达式，并基于3GPP标准信道模型进行端到端仿真，确保结论的工程指导意义。

🔄 替代方案
LEO卫星直连IoT 利用低轨卫星直接服务IoT设备，覆盖范围更广但时延和成本较高，适合对延迟不敏感的大规模广域监测场景。
UAV集群协同中继 多UAV形成集群协同中继，通过强化学习优化飞行轨迹和资源分配，覆盖能力更强但引入编队控制和协同管理的额外复杂性。
边缘计算增强UAV 在UAV上部署边缘计算能力，在本地处理部分数据减少回传压力，但受限于机载计算资源和能耗约束。
🔧 核心方法
  • 多 UAV 中继架构 构建多无人机作为 NB-IoT 用户设备与地面基站间中继的非地面网络（NTN）架构，扩展 NB-IoT 覆盖范围至偏远和难以部署区域
  • OMA/NOMA 调度对比 系统性地对比正交多址接入（OMA）和非正交多址接入（NOMA）两种调度策略在多 UAV-NB-IoT 场景中的性能表现
  • LTTF 动态分组 提出基于最长处理时间优先（LTTF）算法的 NOMA 动态分组策略，根据实时负载动态调整用户分组，相比静态分组显著降低时延
  • 闭合式时延表达式 推导了离散均匀分布、泊松分布和 Beta 分布三种流量模型下平均传输时间的闭合式解析表达式，为系统性能评估提供理论工具
  • 3GPP 合规仿真 基于 3GPP 标准信道模型进行端到端物理层仿真，确保实验结果与实际系统标准兼容，增强结论的工程指导意义

📝 中文摘要
展开 / 收起摘要

非地面网络（NTN）在物联网中的应用已成为 5G-Advanced 和 6G 研究的重要方向。3GPP 已将 NB-IoT 确定为 NTN-IoT 的候选技术方案。本文研究了一种基于多无人机的 NB-IoT 非地面通信系统，其中多架无人机在用户设备与地面基站之间充当中继节点。研究系统性地对比了 OMA 和 NOMA 两种多址调度策略，并在 NOMA 框架下提出了静态分组和基于 LTTF（最长处理时间优先）算法的动态分组两种策略。作者推导了在离散均匀分布、泊松分布和 Beta 分布三种流量模型下平均传输时间的闭合式解析表达式。基于 3GPP 标准信道模型的端到端物理层仿真验证了理论分析的正确性。结果表明，NOMA 结合动态分组策略在降低时延和节省能耗方面具有显著优势。

📊 趋势洞察与前瞻分析

一、LLM 在边缘推理与任务卸载中的应用前景

从 Paper 1（VFC 缓存决策）和 Paper 2（MEC 任务卸载）可以看出，LLM 正在从传统的 NLP/CV 领域快速渗透到边缘计算的资源管理与决策优化中。这一趋势的核心驱动力在于：(1) LLM 具备对复杂系统状态的语义理解能力，能够处理传统优化方法难以建模的多源异构信息；(2) 基于上下文推理的"免重训练"特性天然适配边缘网络拓扑动态变化的场景；(3) GRPO 等强化学习微调方法使 LLM 能够获得长期规划能力，克服了 SFT 的短视问题。预计未来 1-2 年内，LLM-as-Agent 将成为边缘计算智能决策的主流范式之一，尤其在车联网、工业互联网等对实时性和自适应性要求高的场景中。

二、Foundation Model 在无线通信中的落地路径

Paper 3（SoM 框架）展示了一条清晰的"基础模型 + 无线通信"落地路径：从通用 LLM 的跨域迁移（LLM4PG/LLM4SG 用于信道参数生成）到领域专用基础模型的端到端设计（WiCo/WiFo/WiPo 覆盖信道建模到协作感知全链路）。这预示着无线通信正在经历从"模型驱动"到"数据驱动基础模型"的范式转换。对于 VEC 而言，这意味着未来的 V2X 通信系统可能不再依赖于预设的信道模型和协议栈，而是由经过预训练的无线基础模型自适应地管理频谱资源、调制编码和协作感知策略。这一转变将大幅提升 VEC 系统在复杂城市环境中的鲁棒性和效率。

三、边缘基础设施的多元化与协同

Paper 4（多 UAV-NB-IoT）虽然未直接涉及 LLM，但其展示的多 UAV 协同中继架构代表了边缘基础设施多元化的发展方向。结合 Paper 1 的 VFC 概念和 Paper 2 的 MEC 框架，可以预见未来 VEC 的边缘计算节点将包括：路侧单元（RSU）、车载雾节点（VFC）、无人机中继（UAV）和低轨卫星（LEO）等多种形态。如何在这些异构节点之间实现智能协同，将是 LLM/Foundation Model 在 VEC 领域的重要应用方向。

🔬 关键趋势 1：LLM 替代 DRL

LLM + RLHF/GRPO 正在替代传统 DRL 成为边缘决策的新范式。核心优势在于泛化能力和语义理解，解决了 DRL 拓扑敏感和需频繁重训的痛点。

🔬 关键趋势 2：无线基础模型

领域专用基础模型（如 WiCo/WiFo）正在重塑无线通信系统设计方法论，从"数学建模 + 优化求解"转向"预训练 + 微调 + 推理"。

🔬 关键趋势 3：免重训练泛化

多篇论文强调"无需重训练即可适应新场景"，这是基础模型在边缘计算中落地的核心竞争力，显著降低部署和维护成本。

🔬 关键趋势 4：多模态融合

从单一信号处理到感知-通信-计算一体化，基础模型使多模态信息融合成为可能，推动 6G VEC 系统向 AI 原生架构演进。

🔬 关键趋势 5：标准化驱动

3GPP NTN 标准和 NB-IoT 演进为边缘大模型的实际部署提供了标准化接口和协议保障，加速技术从实验室到产业的转化。

🔬 关键趋势 6：协同智能

从单节点智能到多层级协同智能（车-VFC-云 / RSU-UAV-卫星），LLM 的推理能力使跨域协同决策成为可能。
